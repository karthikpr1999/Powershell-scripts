## Running Scripts

```powershell
# Check/set execution policy if scripts are blocked
Get-ExecutionPolicy
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

# Run any script from the repo root
.\ScriptName.ps1

# Run with administrator privileges (required for VPN-HealthCheck.ps1 and UDP port tests)
Start-Process powershell -Verb RunAs -ArgumentList "-File .\VPN-HealthCheck.ps1"

# test1.ps1 — pass gateway IP via argument or environment variable
.\test1.ps1 -GatewayIP "x.x.x.x"
$env:VPN_GATEWAY_IP = "x.x.x.x"; .\test1.ps1

# Hourly_LM_email.ps1 — set LM URL via environment variable or config file
$env:LM_URL = "https://your-account.logicmonitor.com/..."; .\Hourly_LM_email.ps1
```

No build step, no external modules, no package manager. All scripts use only native Windows tools and .NET APIs available in PowerShell 5.1+.

## Architecture

**No shared modules or dot-sourced files.** Every script is fully self-contained. There is no common library, no imports between scripts.

### Script categories

| Category | Scripts |
|---|---|
| Full VPN diagnostic (HTML + JSON reports) | `VPN-HealthCheck.ps1` |
| Quick connectivity checks (console only) | `Pulse_Check_network.ps1`, `gateway_external_link_test.ps1`, `Gateway_internal_link_test.ps1` |
| Interactive user-input diagnostics | `user_input_link.ps1`, `test1.ps1` |
| Security tooling | `Random_password_generator.ps1` |
| Recurring reminder/monitor loops | `ticket_remainder.ps1`, `Hourly_LM_email.ps1` |
| VPN UDP port test | `test.ps1` |

### VPN-HealthCheck.ps1 — key design

The most complex script. Its structure:
1. **Config loading** — reads `VPN-HealthCheck.config.json` (persisted from prior run); prompts for missing values and saves them back
2. **Parallel test execution** — uses a `RunspacePool` to run DNS, ping, TCP, HTTPS, traceroute, and VPN adapter detection concurrently
3. **Result accumulation** — each test calls `Add-Result` to append a `PSCustomObject` (`Test`, `Status`, `Detail`, `DurationMs`) to a shared `$Results` list
4. **Report generation** — produces both `Reports/VPN-HealthCheck-<timestamp>.html` (styled pass/fail table) and `Reports/VPN-HealthCheck-<timestamp>.json` at the end

### Configuration pattern

Scripts that need environment-specific values follow this pattern:
- **`VPN-HealthCheck.ps1`**: prompts on first run, persists to `VPN-HealthCheck.config.json` (gitignored), reloads on next run
- **`Hourly_LM_email.ps1`**: reads `$env:LM_URL` first, falls back to `Hourly_LM_email.config.json` (gitignored)
- **`test1.ps1`**: reads `$env:VPN_GATEWAY_IP` or `-GatewayIP` parameter

Use `*.config.example.json` files as templates — the real `*.config.json` files are gitignored.

### Input validation pattern

Scripts that accept user input (`user_input_link.ps1`, `test1.ps1`, `Pulse_Check_network.ps1`) validate before passing to native commands:
- Hostname/IP inputs are checked against `'^[a-zA-Z0-9.\-:]+$'`
- MTU is validated as integer in range 68–9000
- Gateway IP is validated against IPv4 format regex

### Subdirectories

- `Internal_external_user_link_testing/` — development variants of the gateway/user-input scripts; not the canonical versions
- `Powershell-scripts/` — duplicate tree of most root scripts; treat as secondary copies, root is authoritative
- `Reports/` — auto-generated by `VPN-HealthCheck.ps1`; gitignored
- `logic_monitor_logs/` — append-only log from `Hourly_LM_email.ps1`; gitignored
